原因

Function String.matchAll() is supported in Node.js from version 12.0.0, so, upgrade your nodejs to a version greater than 12.0.0.

OR

Changing the value of config item highlight->enable in file _config.xml from true to false, this can avoid the exception.

bovensonAnswer

意思为：

版本12.0.0的Node.js中支持String.matchAll()

两种解决方法:

  1. 请将nodejs升级到高于12.0.0的版本

这样可以避免异常，但方法二会让代码失去高亮显示功能

SourceTree的问题

问题

在说明这个问题之前，我是用的是最新版的SourceTree（v4.2.0），在感叹它好用之余，就不行被坑到了

在我成功构建之后，我代码仓库中的代码就会自动被编译好，也就是说，我的代码仓库跟我本地电脑的内容是不一样的

这时候我再push我的hexo源码去我的代码仓库就会出现以下报错

意思为

更新被拒绝，因为您当前分支的最新提交落后于其对应的远程分支。

再次推送前，先与远程变更合并（如 ‘git pull …’）

解决方法

解决方案是需要先获取远端更新并与本地合并，再git push，但这个方法不适合去解决这个问题

这个时候必须要用到SourceTree里的强制推送功能

很坑的是SourceTree v4.x.x使用无法直接开启强制推送功能，而v4.x.x则可以开启，因此，直接选择v3.x.x的SourceTree，我这里使用的是v3.2.1(255)，这个问题就迎刃而解了
